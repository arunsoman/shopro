# AI-Powered Sales Analytics Requirements

## 1. Overview
This document captures unambiguous User Stories for the AI-Powered Sales Analytics module. This feature provides restaurant owners and managers with deep, actionable insights into historical performance, automated trend spotting, and predictive forecasting to optimize labor and revenue.

## 2. User Roles
*   **Owner/General Manager:** The primary user who relies on top-level dashboards to make strategic business decisions, create schedules, and analyze profitability.
*   **System (AI Engine):** Processes historical transaction data to generate summaries, predictive text outputs, and visual charts.

## 3. User Stories

### Epic 1: Comprehensive Data Visualization
**Goal:** Provide clear, immediate visualizations of key performance indicators (KPIs) over user-defined time periods.

*   **US-1.1: Customizable Dashboard Views**
    *   **As an** Owner, **I want to** select a date range (e.g., "Today", "Last 7 Days", "Custom Date Range") from a dropdown, **so that** all charts and metrics on the dashboard instantly update to reflect that period.
    *   *Acceptance Criteria:* The dashboard must recalculate all values within 2 seconds of a date range change. Default view upon logging in must be "Current Day vs. Same Day Last Week".
    *   **Entities:** `DailySalesSnapshot`
    *   **Tech Stack:** React + shadcn + Tailwind
*   **US-1.2: Gross Sales and Labor Cost Overlay**
    *   **As an** Owner, **I want to** view a line graph overlaying hourly gross sales against hourly labor costs, **so that** I can identify periods where I am overstaffed relative to incoming revenue.
    *   *Acceptance Criteria:* The graph must pull time-clock data natively if integrated, or allow manual entry of aggregate hourly labor cost. The Y-axis represents Dollar Amount, and the X-axis represents Hours of Operation.
    *   **Entities:** `DailySalesSnapshot`, `StaffMember`
    *   **Tech Stack:** React + shadcn + Tailwind
*   **US-1.3: Top Selling Items (Product Mix)**
    *   **As a** Manager, **I want to** view a ranked list of my Top 10 selling menu items by volume and by gross profitability, **so that** I know which dishes are driving my business.
    *   *Acceptance Criteria:* The list must clearly differentiate between "Quantity Sold" (volume) and "Total Margin" (Profitability = Selling Price - Food Cost from Inventory module).
    *   **Entities:** `DailySalesSnapshot`, `OrderItem`, `MenuItem`, `RawIngredient`, `Recipe`
    *   **Tech Stack:** React + shadcn + Tailwind

### Epic 2: AI-Driven Insights & Forecasting
**Goal:** Move beyond static reporting to provide proactive, actionable recommendations generated by an AI analysis engine.

*   **US-2.1: Automated Trend Summaries**
    *   **As an** Owner, **I want the** system to generate a short, natural language summary of my weekly performance (e.g., *"Sales are up 12% compared to last week, driven primarily by a surge in Appetizer orders on Friday night"*), **so that** I don't have to manually interpret complex charts.
    *   *Acceptance Criteria:* The text summary must highlight statistically significant deviations (>10% variance) from historical averages. It must be generated automatically at the close of the business week.
    *   **Entities:** `AIInsight`, `DailySalesSnapshot`
    *   **Tech Stack:** React + shadcn + Tailwind
    *   **Tech Stack:** React + shadcn + Tailwind
*   **US-2.2: Predictive Sales Forecasting (Scheduling Aid)**
    *   **As a** Manager, **I want the** system to predict next week's daily gross sales based on historical data, upcoming local holidays, and weather forecasts, **so that** I can accurately schedule my staff levels.
    *   *Acceptance Criteria:* The forecast must provide a numeric projection (e.g., "$5,400 Expected") for a given future date. The UI must display a "Confidence Score" (e.g., 85% accurate) alongside the prediction.
    *   **Entities:** `AIInsight`, `DailySalesSnapshot`
    *   **Tech Stack:** React + shadcn + Tailwind
    *   **Tech Stack:** React + shadcn + Tailwind
*   **US-2.3: Promotion and Menu Optimization Suggestions**
    *   **As an** Owner, **I want the** AI to identify slow periods or underperforming items and suggest targeted actions (e.g., *"Suggestion: Implement a Happy Hour promotion between 4 PM and 6 PM on Tuesdays to boost low historic volume"*), **so that** I can act immediately to increase revenue.
    *   *Acceptance Criteria:* Suggestions must be presented in a dedicated "Actionable Insights" widget. They must tie directly to actionable POS features (like generating a timed discount code if the user clicks "Apply Suggestion").
    *   **Entities:** `AIInsight`, `DailySalesSnapshot`, `MenuItem`
    *   **Tech Stack:** React + shadcn + Tailwind

## 4. Ambiguity Review Summary
*   **Latency Expectations (US-1.1):** Specified a 2-second recalculation constraint so developers understand the dashboard must be responsive, requiring optimized data aggregation queries rather than raw row counting on the fly.
*   **Profitability Definition (US-1.3):** Clarified that "Profitability" is a calculated value inextricably linked to the Food Cost data derived from the Inventory module (US-1.2 of the Inventory spec), establishing a strict data dependency between the two modules.
*   **AI Predictability (US-2.2):** Explicitly required a "Confidence Score" alongside predictive forecasting so users are aware of the potential variance, converting an ambiguous "guess" into a statistically weighted metric.

## 5. Gap-Resolved Stories

### Epic 3: Data Export & Reporting
**Goal:** Allow Owners and Managers to export analytics data for external use in accounting, payroll, and reporting tools.

*   **US-3.1: Exporting Dashboard Data to CSV**
    *   **As an** Owner, **I want to** export the raw data behind the current dashboard view as a CSV file, **so that** I can import it into accounting or payroll software for further analysis.
    *   *Acceptance Criteria:* The "Export CSV" button must respect the currently active date range filter. The CSV must include one row per transaction, with columns for: Date, Time, Order ID, Table, Server, Items (comma-separated), Subtotal, Discount, Tax, Tip, Total, and Payment Method. The export must be generated and begin downloading within 5 seconds for date ranges up to 90 days. For ranges exceeding 90 days, the system must display: "Large export in progress. You will be notified when the file is ready."
    *   **Entities:** `OrderTicket`, `Payment`, `DailySalesSnapshot`, `StaffMember`
    *   **Tech Stack:** React + shadcn + Tailwind

*   **US-3.2: Exporting a Dashboard View to PDF**
    *   **As an** Owner, **I want to** export the current analytics dashboard as a PDF report, **so that** I can share a visual summary with my business partners or bank without requiring them to log in to the POS.
    *   *Acceptance Criteria:* The PDF export must include all charts and KPI tiles visible in the current dashboard view at the time of export. The PDF header must include the restaurant name, the report date range, and the generation timestamp. Charts must be rendered as vector graphics (not screenshots) for print quality. The PDF must be generated and downloadable within 10 seconds.
    *   **Entities:** `DailySalesSnapshot`, `AIInsight`
    *   **Tech Stack:** React + shadcn + Tailwind

### Epic 4: Table & Floor Optimization Analytics
**Goal:** Optimize dining room layout and server performance through spatial data analysis.

*   **US-4.1: Table Utilization Dashboard**
    *   **As a** Manager, **I want to** see which tables are most/least utilized, **so that** I can optimize floor plan and pricing.
    *   *Acceptance Criteria:* Metrics: Covers per day, Revenue per seat, Turn rate, Average party size, Idle time %. Heat map overlay on floor plan: Green (high util) to Red (low util). Comparison: Weekday vs Weekend.
    *   **Entities:** `TableShape`, `TableAnalytics`, `RevenueData`
    *   **Tech Stack:** React + shadcn + Tailwind
*   **US-4.2: Table Revenue Attribution**
    *   **As an** Owner, **I want to** know exactly how much revenue each table generated, **so that** I can make data-driven decisions about seating priorities.
    *   *Acceptance Criteria:* Net revenue per table (after comps/discounts). Per-seat revenue comparison. Table-type analysis: Booth vs Standard vs Patio vs Bar. Time-series: Revenue by table by hour.
    *   **Entities:** `TableShape`, `OrderTicket`, `Payment`
    *   **Tech Stack:** React + shadcn + Tailwind

### Epic 5: Menu Engineering & PMIX Analysis
**Goal:** Optimize the menu based on profitability and popularity data.

*   **US-5.1: Menu Engineering Matrix (Stars/Plowhorses/Puzzles/Dogs)**
    *   **As an** Owner, **I want to** view my menu items categorized into a 4-quadrant matrix based on Popularity (Volume) and Profitability (Margin), **so that** I can decide which items to promote, rework, or remove.
    *   *Acceptance Criteria:* Matrix display:
        *   **Stars:** High Profit, High Popularity (Maintain).
        *   **Plowhorses:** Low Profit, High Popularity (Increase price/rework recipes).
        *   **Puzzles:** High Profit, Low Popularity (Market/Promote).
        *   **Dogs:** Low Profit, Low Popularity (Remove from menu).
    *   **Entities:** `MenuItem`, `Recipe`, `DailySalesSnapshot`
    *   **Tech Stack:** React + shadcn + Tailwind

### Epic 6: Intelligent Surveillance & Anomaly Detection
**Goal:** Use AI/ML to identify suspicious patterns, theft, and operational waste.

*   **US-6.1: High-Risk Action Anomalies**
    *   **As a** GM, **I want the** system to flag servers with unusually high rates of voids after payment or excessive comp usage, **so that** I can identify potential theft or training gaps.
    *   *Acceptance Criteria:* Real-time alerts for "Comp rate > 10% of sales". Benchmarking server performance against peers. 
    *   **Entities:** `AuditLog`, `StaffMember`, `DailySalesSnapshot`
    *   **Tech Stack:** React + Backend AI Engine
*   **US-6.2: Inventory Shrinkage Sentiment Analysis**
    *   **As an** Owner, **I want the** system to correlate inventory waste reasons (e.g., "Dropped Plate") with staff performance and peak hours, **so that** I can identify root causes of shrinkage.
    *   *Acceptance Criteria:* Heatmap showing waste events over time and staff.
    *   **Entities:** `InventoryTransaction`, `WasteReason`, `DailySalesSnapshot`
    *   **Tech Stack:** React + shadcn + Tailwind

### Epic 7: Executive GM Dashboard
**Goal:** Provide a high-level operational pulse for restaurant leadership.

*   **US-7.1: Real-time P&L & Labor Alerting**
    *   **As a** General Manager, **I want a** mobile-optimized dashboard showing my current Daily P&L and Labor % (Labor vs. Net Sales), **so that** I can adjust staffing mid-shift.
    *   *Acceptance Criteria:* Color-coded alerts: Green (<25% labor), Yellow (25-30%), Red (>30%). Real-time feed from POS and Clock-In system.
    *   **Entities:** `StaffShift`, `OrderTicket`, `Payment`
    *   **Tech Stack:** Flutter (Manager App) / React (Admin)
